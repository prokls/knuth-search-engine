Syntax

The following examples illustrate normal search patterns. You can combine them by concatenating them with a space. The first search pattern gets more importance than the last one.

word Search for "word"
/word/ Search with Regular Expression "word" (allowed modifiers: ixU). Only / as delimiter accepted.
word1 word2 Search for "word1" or "word2"
word1 + word2 Search for "word1" and "word2"
"word" Search for exact phrase "word"

Search using fields

A field consists of a key-value-pair in a notation " key:value". An example is " lang:de"; searching only for german documents. If the value consists of multiple words, you have to use quotes (', ") to combine them. An example is " author:'Stu dent'". Attention! Quotes for a single word will be an "exact phrase match" (eg. " author:'student'").

Search for meta data

key value description
lang The language in ISO 639-1:2002 as a string. Eg. "en" or "de"
course A string representing the title of a course (or each first letter of a title). Eg. "GDI"
author A string naming the author. Eg. "meisterluk"
license A string describing the license of the document (or License-ID; see upload page HTML source). Eg. "GPL"
Note: Only works good for main software licenses (GPL, BSD) and Creative Commons
file The filename of the main document

Search with general information

key value description
filetype A string describing the file extension (mostly three-letters string). Eg. "txt" or "tex"
assoc A boolean value, stating existence of an associated document

Specify appearence

key value description
list An integer (between 1 and 30) specifying the number of displayed search matches. Defaults to 12.
page An integer (greater than 0) specifying the ID for results page. Eg. " list:10 page:2" displays matches #11 to #20. Defaults to 1.
